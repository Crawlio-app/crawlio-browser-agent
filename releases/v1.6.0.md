# v1.6.0

**Date:** 2026-03-11

## Fixes

### CWS Extension Bridge Connection

The Chrome Web Store extension could not connect to the MCP server due to Chrome's Private Network Access (PNA) policy blocking health probe requests from the extension to localhost.

### Changed

- Added `Access-Control-Allow-Private-Network: true` header to bridge health endpoint and OPTIONS preflight
- Added `Access-Control-Allow-Headers: *` to OPTIONS preflight for PNA compatibility
- Reconnect alarm now created at start of `discoverAndConnectAll()` — ensures persistent retry even when all initial probes fail
