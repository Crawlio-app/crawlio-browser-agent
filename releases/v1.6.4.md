## Browser Agent v1.6.4

### Platform Thesis Alignment

The extension is now a dumb CDP bridge — no intelligence, no passive detection, no badges. Per the platform thesis: "The extension is Docker. It runs containers. It doesn't know what's in them."

- **Heartbeat badge removed** — no more passive framework/tracking pixel counting on the extension icon
- **SERP auto-detection removed** — SERP tools still work via MCP, but the extension no longer auto-badges or auto-injects overlays
- **Auto-connect on wake** — extension probes for MCP server on `onStartup`, `tabs.onActivated`, `tabs.onUpdated`. No popup "Connect" click required.

### CDP Stability

- **Recovery backoff** — CDP domain re-enable retries 3 times with [1s, 3s, 5s] timeouts instead of single-pass failure. Heavy SPAs (Linear, Vercel) no longer cause "recovery failed" on navigation.
- **DOM query timeout** — `DOM.getDocument` bumped from 5s to 10s for accessibility snapshots on React/Next.js sites.

### Evidence Mode Skills

All 9 skills rewritten for the full JIT Context stack — Code Mode + Method Mode + Evidence Mode:

- **browser-automation** — reference skill with Evidence Protocol, Anti-Patterns, 24 smart methods documented
- **web-research** — Acquire → Normalize → Analyze with smart.finding() and confidence propagation
- **investigate** — deep-dive single-site analysis with 6 dimension findings
- **extract** — structured data extraction with data-quality findings
- **compare** — side-by-side comparison across 11 scaffold dimensions
- **clone** — design system capture with CSS custom property extraction
- **dossier** — competitive intelligence across N competitors
- **monitor** — change detection with diffSnapshots
- **test** — quality assertions (accessibility, performance, security, SEO, mobile)

### Execute Sandbox

- **ocrScreenshot()** added as 7th sandbox parameter — macOS Vision.framework OCR accessible from code mode
- **Anti-patterns** added to execute tool description — warns about smart.screenshot() (doesn't exist), sleep loops, location.href navigation, smart.snapshot({compact}) (doesn't exist)
